## Overview

The **Event Processor Service** is a .NET 9 Worker Service that acts as the event-driven backbone of the platform. It reads domain events from the Outbox table and routes them to downstream consumers.

**Actors:** Internal only — no direct human interaction.

**Key features:**
- Outbox pattern consumer (polls and dispatches domain events)
- Event bus integration (RabbitMQ, Kafka, NATS)
- Retry logic and Dead Letter Queue (DLQ)
- Event enrichment (adds metadata before forwarding)
- Audit logging of all processed events
